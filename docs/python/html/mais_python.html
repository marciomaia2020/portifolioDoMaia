Configuração do Perfil PowerShell
Exibe o caminho do arquivo de perfil atual do PowerShell, onde são armazenadas as configurações personalizadas e funções do PowerShell. echo $PROFILE
Caminho padrão onde o perfil do PowerShell é armazenado no sistema Windows. Este é o local onde todas as configurações e funções personalizadas são mantidas. %SystemRoot%\Users\Documents\WindowsPowerShell\Microsoft.PowerShell_profile.ps1
Cria um novo arquivo de perfil do PowerShell se ele ainda não existir. Usa o parâmetro -Force para garantir a criação mesmo se já existir. New-Item -ItemType File -Path $PROFILE -Force
Abre o arquivo de perfil do PowerShell no Notepad para edição, permitindo adicionar ou modificar configurações. notepad $PROFILE
Define uma função para ativar o ambiente virtual, verificando se o caminho existe antes de tentar a ativação. Se o caminho não for encontrado, exibe uma mensagem de erro em vermelho. function activate { param([string]$path = ".\venv\Scripts\Activate") if (Test-Path $path) { & $path } else { Write-Host "O caminho para o ambiente virtual não foi encontrado: $path" -ForegroundColor Red }}
Recarrega o perfil do PowerShell para aplicar as alterações feitas, tornando as novas configurações e funções disponíveis na sessão atual. . $PROFILE
Exibe a lista completa das políticas de execução no PowerShell, mostrando as configurações para cada escopo. Get-ExecutionPolicy -List
Altera a política de execução para permitir scripts não assinados no PowerShell, aplicando apenas para o usuário atual por questões de segurança. Set-ExecutionPolicy Unrestricted -Scope CurrentUser

Ambiente Virtual Python
Cria um ambiente virtual Python padrão na pasta atual, isolando as dependências do projeto. python -m venv venv
Cria um ambiente virtual específico para Python (CPython), permitindo separar claramente de outros ambientes. python -m venv venv-python
Cria um ambiente virtual específico para PyPy3, otimizado para performance com o interpretador PyPy. pypy3 -m venv venv-pypy
Ativa o ambiente virtual Python (CPython), tornando disponíveis os pacotes e configurações específicas deste ambiente. .\venv-python\scripts\activate
Ativa o ambiente virtual PyPy3, permitindo usar o interpretador PyPy e suas otimizações. .\venv-pypy\scripts\activate
Remove completamente um ambiente virtual e todos seus arquivos de forma forçada, liberando espaço no disco. Remove-Item -Recurse -Force .\venv

Configuração e Uso do Redis
Inicia o servidor Redis no Windows. É necessário estar no diretório de instalação ou ter o Redis configurado nas variáveis de ambiente do sistema. redis-server.exe
Testa a conexão com o servidor Redis. Se a conexão estiver funcionando corretamente, o comando retornará "PONG". redis-cli ping
Configuração de Logging em Python
Importa o módulo de logging no Python para configurar o sistema de registro de eventos e mensagens do aplicativo. import logging
Configura as opções básicas do logging, incluindo nível, formato da mensagem e handlers para arquivo e console. logging.basicConfig(level=logging.INFO, format='%(asctime)s - %(levelname)s - %(message)s', handlers=[logging.FileHandler("app.log"), logging.StreamHandler()])
Registra uma mensagem informativa no log, útil para acompanhar o fluxo normal do programa. logging.info()
Registra uma mensagem de aviso no log, indicando situações potencialmente problemáticas. logging.warning()
Registra uma mensagem de erro no log, utilizada quando ocorrem problemas que precisam de atenção. logging.error()

Git e Controle de Versão
Verifica se um arquivo específico está sendo ignorado pelo Git e mostra qual regra do .gitignore está causando isso, útil para debug de configurações do Git. git check-ignore -v [arquivo]
Remove um arquivo específico do cache do Git mantendo-o no sistema de arquivos, útil para começar a ignorar um arquivo que já estava sendo rastreado. git rm --cached [arquivo]
Gerenciamento de Pacotes Python
Garante que o pip está instalado no sistema Python atual. Se não estiver instalado, realiza a instalação automaticamente. python -m ensurepip
Atualiza o pip para a versão mais recente, garantindo acesso às últimas funcionalidades e correções de segurança. python -m pip install --upgrade pip
Instala o pacote python-dotenv para gerenciamento de variáveis de ambiente através de arquivos .env, útil para configurações sensíveis. pip install python-dotenv

Descrição Operador Exemplo
Versão maior que a especificada. Útil quando você precisa de uma versão mais nova que determinado ponto. > >3.1
Versão menor que a especificada. Use quando precisa garantir compatibilidade com versões anteriores. < <3.1
Versão maior ou igual à especificada. Bom para garantir funcionalidades mínimas necessárias. >= >=3.1
Versão menor ou igual à especificada. Útil para evitar breaking changes em versões mais novas. <= <=3.1
Exatamente a versão especificada. Use quando precisa garantir total compatibilidade com uma versão específica. == ==3.1
Versão diferente da especificada. Útil quando uma versão específica tem um bug conhecido. != !=3.1
Versão compatível (permite atualizações do último segmento). Similar ao >=3.1,==3.1.* ~= ~=3.1.2
Coringa, corresponde a qualquer versão. Permite qualquer versão no segmento especificado. * ==3.1.*
