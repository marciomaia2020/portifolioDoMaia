Fundamentos de web services

Os Fundamentos de web services - Envolvem a comunicação entre sistemas através da internet, permitindo que diferentes aplicações se interconectem e troquem informações. Alguns conceitos importantes incluem:

  1. Protocolos de comunicação: Os web services geralmente usam protocolos como HTTP (Hypertext Transfer Protocol) para enviar e receber dados.
  2. Formatos de dados: Os dados são geralmente trocados em formatos como XML (Extensible Markup Language) ou JSON (JavaScript Object Notation).
  3. APIs (Application Programming Interfaces): As APIs definem os métodos e estruturas de dados que os web services expõem para permitir a interação com outros sistemas.
  4. SOAP (Simple Object Access Protocol): É um protocolo baseado em XML usado para troca de mensagens entre sistemas distribuídos.
  5. REST (Representational State Transfer): É um estilo arquitetural que define princípios para projetar serviços web, utilizando o protocolo HTTP e recursos identificados por URLs.
  6. WSDL (Web Services Description Language): É uma linguagem baseada em XML que descreve a interface de um serviço web, incluindo os métodos disponíveis e seus parâmetros.

APIs REST

APIs REST (Representational State Transfers) - São um estilo arquitetural para o desenvolvimento de serviços web. Elas permitem a comunicação entre sistemas através do uso de requisições HTTP, seguindo princípios como statelessness, cacheability e uma interface uniforme. As APIs REST são amplamente utilizadas para criar aplicações web e móveis que se integram com outros sistemas.

JSON Web Tokens

JSON Web Tokens (JWT) - São um padrão aberto (RFC 7519) para token de acesso baseado em JSON. Eles são usados para autenticação e troca de informações entre partes confiáveis. Um JWT é composto por três partes: o cabeçalho (header), a carga útil (payload) e a assinatura. O cabeçalho contém informações sobre o tipo de token e o algoritmo de assinatura usado. A carga útil contém os dados adicionais que são transmitidos junto com o token, como informações de usuário ou permissões. A assinatura é usada para verificar a integridade do token. Os JWTs são amplamente utilizados em APIs para autenticação e autorização de usuários.

Swagger

Swagger - É uma ferramenta de código aberto que permite documentar, testar e visualizar APIs REST. Ele fornece uma interface interativa onde os desenvolvedores podem explorar os endpoints, ver exemplos de requisições e respostas, e até mesmo testar as chamadas diretamente na documentação.
