Trabalho Contínuo com Git

Guia desenvolvido para facilitar o trabalho contínuo entre máquinas usando Git.

Passo 1: Configure o Repositório Remoto em Ambas as Máquinas

Clone o repositório remoto nas duas máquinas: (GITHUB DESKTOP)

No computador 1:

git clone <URL-do-repositório>

No computador 2:

git clone <URL-do-repositório>

Isso garante que ambas as máquinas estão sincronizadas com a versão inicial do repositório remoto.

Confirme a URL remota configurada: Em ambos os computadores, use:

git remote -v
git status
git remote add origin 
git remote -v(se não exibir nada então - Sigano Passos para Vincular um Repositório Remoto)

Isso deve mostrar a URL do repositório remoto (como no GitHub ou GitLab). Certifique-se de que seja o mesmo em ambas as máquinas.


Passos para Vincular um Repositório Remoto

Se o comando git remote -v não exibiu nada, significa que o repositório local não está vinculado a um repositório remoto. Para resolver isso:

1. Verifique se você está no diretório correto:

Certifique-se de que está no diretório do projeto que contém o repositório Git. Use:

git status

Se a resposta for algo como "fatal: not a git repository," você precisa inicializar ou navegar até o repositório correto.

2. Adicione o repositório remoto:

Se você já possui a URL do repositório remoto, use o comando abaixo:

git remote add origin <URL_DO_REPOSITORIO_REMOTO>

Exemplo para GitHub:

git remote add origin https://github.com/usuario/nome-repositorio.git

3. Confirme que o repositório remoto foi adicionado:

Execute novamente:

git remote -v

Agora, o comando deve mostrar algo como:


origin  https://github.com/usuario/nome-repositorio.git (fetch)
origin  https://github.com/usuario/nome-repositorio.git (push)

4. Empurre o código para o repositório remoto (se necessário):

Caso o repositório remoto esteja vazio, envie o código:

git push -u origin main

Nota: Substitua main pelo nome do branch principal do seu repositório, se for diferente.

Passo 2: Trabalhe Localmente e Sincronize com o Repositório Remoto

Antes de começar a trabalhar, sincronize com o remoto: (GITHUB DESKTOP)

git pull origin <branch>

Isso garante que você está trabalhando com a versão mais atualizada.

Faça suas alterações normalmente:


	git add .
             git commit -m "Descrição das alterações"
     (GITHUB DESKTOP)

Envie as alterações para o repositório remoto: (GITHUB DESKTOP)

git push origin <branch>

Passo 3: Sincronize a Outra Máquina

Na outra máquina, antes de começar a trabalhar, sincronize novamente para obter as mudanças feitas:

git pull origin <branch>(GITHUB DESKTOP)

Observação: Nunca esqueça de fazer um Fech Origin (Sincronização com o original) para que o GitHub Desktop faça a sincronização com o repositório original.

Passo 4: Evite Conflitos

Evite trabalhar na mesma branch ao mesmo tempo:

Se ambos os computadores editarem a mesma branch simultaneamente, podem surgir conflitos. Para minimizar isso:

  • Use branches separadas para cada conjunto de alterações (por exemplo, veronica-machine1 e veronica-machine2).

  • Após concluir o trabalho, mescle essas branches no remoto com git merge ou por meio de Pull Requests (em plataformas como GitHub).

    • Resolva conflitos rapidamente: Se ocorrerem conflitos, o Git mostrará os arquivos com problemas. Resolva manualmente, teste o código e finalize o processo:

    
git add <arquivo-com-resolucao>
git commit -m "Resolvido conflito"
git push origin <branch>

    Resumo: Fluxo Geral

    Clonar os repositórios em ambas as máquinas.

    Antes de começar: Sempre execute git pull.

    Depois de trabalhar: Faça git add, git commit e git push.

    Sincronize frequentemente para minimizar conflitos.
